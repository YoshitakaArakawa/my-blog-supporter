# 記事構成: Dataiku Summit 2026 に行って ──「オーケストレーション層の中心に、誰を置くのか」（タイトル仮）

## コアメッセージ
- 「AIエージェントのオーケストレーション層が要るか」では業界は揉めていない。
- 本当の分岐は **「その層の中心に人間を置くか、エージェントを置くか」（center軸）**。
- 自分個人の物差しはさらに別の軸 ──**「開いた土台（コード）か、閉じたプラットフォーム（機能の枠）か」（substrate軸）**。
- 二軸は今は相関するが同一ではない。自分は「開いた土台」に賭けているが、相関が続くのか・ほどけるのかは分からない。だから値踏みせず観測し続ける。

## 読後の読者の状態
- AIエージェントを現場で触り始めた読者（＋Tableauユーザー）が「腑に落ちる・自分も辿り直したくなる」。
- 特に、自分の引っかかりの物差しが "開/閉" なのか "人間/エージェント" なのかを切り分けて考え直したくなること。
- 煽り・断定でなく、再現可能な思考の筋で。

## トーンの約束（全セクション通底）
- Dataiku（と中の人）へのリスペクト前提。製品の値踏みではなく、自分の現在地からの見え方を晒す。
- バイアス（substrate軸）は早く開示し、steelman とセットでのみ使う。
- 「分からない」は逃げにしない（暫定の賭け＋反証条件を出す）。
- 数値・固有名詞は深掘らず自分の言葉で。時刻は「午前/午後/夜」程度にぼかす。
- 本文は内省エッセイ＝地の文（散文）が基本。本 outline の箇条書きは「話す順の骨」であって本文の形ではない。

### 展開1: まず、オーケストレーション層は要る ── そこは自分も同意する（steelman先出し）
展開する思考:
- **先に肯定**: 「層が要るか」と聞かれたら自分も要ると思う。
- **なぜ要るか**: 複数LLM・複数エージェントが別々に動くとき、「何が・誰のために・どう動いているか」を横断して管理・制御する層は必要だと感じる。
- **プロバイダの上の層**: OpenAI/Claude が各々プラットフォームを持つ。その"上"に、特定プロバイダ非依存の別レイヤーがあるべき、は腑に落ちる。
- **問題設定は良い、と明言**: だからこの問題設定そのものは、とても良いと思っている。
- **軽い歴史（一文）**: Dataiku は2014年から視覚フロー（データパイプラインのオーケストレーション）の会社。それをエージェントのオーケストレーションへ向け直した＝思想は一貫、流行りに飛びついたのではない（フェアに）。
- **ブリッジ**: ここまでは素直に同意できる。引っかかるのはこの先だった。
素材: discussion-root「Dataiku 自身の発信①」「Dataiku製品史」、公式メッセージ（NI＋C／DCS）。
接続: 引っかかりの前半＝肯定を先に置き、後の懐疑を値踏みでなく誠実な問いに見せる。

### 展開2: 引っかかりの正体①── 問いを置き換える（center軸＋物差しの開示）【厚く】
展開する思考:
- **問いの再定義**: 「層が要るか」では揉めていない。本当の分岐は **中心に人間かエージェントか**。
  - Dataiku は明確に人間中心（監査・承認・サインオフを"アーキテクチャ要件"に）。
  - それは2014年からのDNAで、日和見ではない（フェア）。
- **自分の物差しを正直に開示＝substrate軸**（開いた土台か、閉じたプラットフォームか）。
  - 「人間の可能性を狭めたくない」の正体＝プラットフォームの機能の枠に実装・活用が縛られること＝**操作者の天井**の問題。
  - 自動化そのものへの反対ではない（敵はプラットフォームの天井であって自動化ではない）。
- **非対称を認める**:
  - center軸＝観測した客観。
  - substrate軸＝自分の主観（n=1）。
  - だからこれは「客観の借用」ではなく、「自分の好みを業界分析から隔離して晒す」開示。
- **暫定の賭けを先出し**: 自分は「開いた土台」に賭けている、と隠さず言う。
- **二軸は相関するが同一でない**（flag1）:
  - 人間中心の事例があること自体は否定しない。
  - 閉じた×エージェント中心も自分は嫌／開いた×人間中心（例: Claude Code を CI に入れ人間が必ず PR レビュー）は許せる。
  - ＝効いているのは "開/閉" の方。
素材: discussion-root「substrate軸×center軸の入れ子」「筆者のレンズ（バイアス開示）」。thinking.md 観点1。
接続: 「必要性は分かる」の次に「では何に引っかかるのか」を問いの再定義で受ける。

### 展開3: 引っかかりの正体②── 同じ2026年、二陣営が正反対に動いている（時間軸の対比）【厚く・山場】
展開する思考:
- **center軸は静的な二分でなく、方向を持つ**。
- **capability陣営（Anthropic 直近半年）は人間確認の価値を"薄め"ている**:
  - harness：Evaluator が人間QAを肩代わり。
  - long-run：4-6h の自律セッション。
  - loop：outcomes の自己採点／dreaming の自己改善。
  - Managed Agents を境に、人間が「ループの中→上→外」へ。
- **語の精度**: 「薄める」であって「消す」ではない（人間はルーブリックを書き Evaluator を調律する役で残る）。
- **同じ時期、Dataiku は逆に人間確認を"太らせて"いる**。
  - ＝二陣営が人間確認をめぐって正反対に動いている。これが一番きれいな対比。
- **自分とAnthropicの関係＝「影響を正直に認める」**:
  - 自分は独立に同じ結論へ至った観察者ではない。Anthropic の articulation に価値観を形成され、大いに参考にしている。
  - 借り物の権威づけを避けるのは独立を装うことでなく、影響を認めること。
  - その上で **自分の付加＝土台の開放性（開/閉）の強調**：Anthropic のモデル能力への賭けに同意しつつ、"開いた土台の上での自律"は歓迎・"閉じたプラットフォーム"は自律でも嫌、を足す。
  - capability-first は自分の整理ラベルで Anthropic の自称ではない、と断る。
  - **[著者がwrite時に一次の言葉で]** どの articulation のどこに・どう影響されたか。
素材: discussion-root「Anthropic のエージェント観の時系列」（原典引用：Building effective agents 2024-12 / multi-agent 2025-06 / harness 2026-03 / Managed Agents 2026-04 / dreaming・outcomes 2026-05）。
接続: 自分の物差し（展開2）を当てると、業界の動き（展開3）がこう見える、という順。

### 展開4: 小さな補助線 ── 視覚フローと、Tableauユーザーとしての揺らぎ【軽く】
展開する思考:
- **視覚フローへの懐疑は (i) ではなく (ii)**:
  - (i) 表現可能性「書けない」── 採らない（分岐ノードがある以上反証される）。
  - (ii) 維持可能性「書けるが複雑さに耐えない」── こちらを採る。
    - ある複雑さ（条件分岐・長い尾のロジック）を超えると、コードより**読めない・直せない・差分が追えない**。
    - テキストの線形性（行diff／grep／版管理との親和）と、視覚の空間配置はトレードオフ。
    - **[本質 or 時限は write 時に確定]** 本質差と言い切る／「現時点の成熟度では」と弱める。
- **射程の限定（自分自身の譲歩として）**:
  - 自分の「開いた土台＝天井が高い」は *技術者にとっての* 天井。
  - 操作者が非技術者（書けないアナリスト）の組織では、コードはむしろ天井を下げうる（書けない人には天井ゼロ）。
  - だから普遍命題ではなく技術者視点に射程を限定する、と自分で正直に降りる。
  - ※この譲歩は他者記事に依らず自分の論で書く（2記事は末尾で紹介）。
- **Tableauユーザーとしての自己観察（読者との接点）**:
  - 自分は Tableau ユーザーでもある。
  - 「視覚的にデータ可視化を作れます」という訴求に対しても、いま"人間が見るための価値"はどこまで強いのか、という同種の揺らぎを持っている。
  - ← Dataiku批評ではなく自己観察に着地。
素材: thinking.md 観点3、discussion-root「視覚フローの価値」。Tableau過去記事リンク（references TODO、write前に1本特定できれば内部リンク）。
接続: 大きな対比（展開3）の後に、身近で具体的な自分の手触りへ降ろしてクールダウン。

### 結び: 分からない。だから観測し続ける
展開する思考:
- **賭けは出した**（開いた土台）。
- **本当に分からないこと**＝center×substrate の相関が今後も続くのか、ほどけるのか。
- **反証条件を置く（逃げにしない）**:
  - 例: 開いた土台で組織展開して統制が破綻し人間中心へ揺り戻す事例を複数見たら、自分の賭けは弱まったと認める。
  - 逆に、閉じた前提の組織が開いた土台に移って天井が上がる事例を見たら、賭けは強まる。
- **観測の派生対象**: 自分の仕事のどれだけがワークフローに落ち、どれだけ落ちないか。
- 残す問い:
  - 問題設定は良い。でも「どのくらい先を見据えた話か」「人間を巻き込んで普及させる道は取るべき道か」。
  - これは自分にも Dataiku にも、業界にもまだ答えが無い。だから観測し続ける。
