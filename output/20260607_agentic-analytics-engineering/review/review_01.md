## 指摘（優先度順）

### 1. 冒頭が「何を書くか」の宣言から始まり、体験・動機の順序が voice-style と逆
- 観点: 文体整合 / 構成・流れ
- 箇所: draft_user.md:5-22
- 問題: voice-style ルール19 は「冒頭は著者の体験・現在地から始め、参考リンク先出し／趣旨宣言／スコープ整理から始めない」「『今回書くこと』の宣言は1-2文に圧縮」を求めている。現状は1行目から「この記事では〜共有します」→ 取り組み2点の箇条書き →（10-12行）年表 →（18-19行）「結果は語りません」のスコープ整理、と宣言とスコープ説明が17行続く。読者が「著者の現在地・なぜ今これをやっているか」を体感する前に、目次的な情報を浴びる構成になっている。AI 的な「まず全体像を提示する」順序に寄っている。
- 提案: 冒頭を「個人 Tableau Cloud 環境で最近こういう実験をしている」という著者の現在地・手触りから始め、取り組み2点の箇条書きと年表は導入の後ろ（または展開の中）に送る。動機（9行「ここ2年ほどで〜主要な要素になっている」）はルール19の言う「執筆動機の簡潔な共有」に当たるので導入に残してよいが、年表3行は本文の「なぜ今」セクション（公式ロードマップの章）に移すと導入が軽くなる。

### 2. 最大の難所（セマンティックレイヤー／Headless BI）が読者前提を超えたまま前置きが薄い
- 観点: 読者難易度 / 読みやすさ
- 箇所: draft_user.md:30-35
- 問題: brief の読者像は「dbt / analytics engineering / セマンティックレイヤー / MCP / text-to-SQL に必ずしも明るくない」層 A（brief.md:16-17）。本文30-31行で「セマンティックレイヤーとHeadless BIの文脈」「分析ロジックを可視化の側から切り離して、一度だけ定義して供給」と概念名が前置きなく出て、参考リンク2点に説明を委ねている。voice-style ルール10は新概念に「（簡単に）」「自分の理解」を添えて1-2文で噛み砕く型を求めるが、ここは噛み砕きが無くリンク頼り。離脱リスクのピークがこの辺りに来ている。
- 提案: 30行の直前に「（簡単に言うと）」レベルで1文、セマンティックレイヤー＝指標やビジネス定義を一か所にまとめて翻訳する層、を著者の言葉で置く。リンクは補足として残す。難所を本筋の手前で軽くする。

### 3. 「動機1／動機2」という見出しが論点要約型で長め、章構成も voice-style の3-4章原則と合わない
- 観点: 文体整合 / 構成・流れ
- 箇所: draft_user.md:23, 36, 55, 76 ほか見出し全般
- 問題: voice-style ルール17は「大見出しは3-4個」「見出しは問い／観察／シンプルな宣言で簡潔に」。現状の見出しは「動機1: Headを軽くしたい」「動機2: データソースの裏側も整えたい」「手作業で整えるのは辛いので、AI Agentにお願いする」など、主張をそのまま見出しにした説明的・やや長めの型。また draft が `#`（H1）と地の文だけで構成され、`##` の章立てと `---` 区切り（ルール11・典型構造）が無い。Wix からの素抜けでマークダウン階層が落ちている可能性がある。
- 提案: outline の章立て（導入／問題提起／見立て／手段／公式と同じ船／結び）に沿って `##` を4章程度に整理し、各見出しを短い宣言・問いに削る（例:「Headを軽くしたい」「データソースの裏側」「手作業は辛い、だからAgent」）。`##` 前後に `---` を入れる。

### 4. 表記ゆれ: 和文と英単語の境界の半角スペース、AI Agent 表記が混在

- 問題: voice-style ルール14は「和文と英単語の境界に半角スペースを入れない（Tableauは、で詰める）」「英語フレーズ内部の語間は残す」「AI Agent 表記に統一」。本文は概ね詰めて書けているが、混在が散見される（例: 「dbtの流儀」「Published Data Source をセマンティックレイヤーとして整備するAI Agent」内で "Published Data Source を" の境界スペースなど、英フレーズ末と和文助詞の間に半角が入っている箇所がある）。「AI Agent」表記は本文で統一できており良い。

### 7. 末尾サインオフが voice-style の固定形と一字違い
- 観点: 文体整合
- 箇所: draft_user.md:82-83
- 問題: voice-style ルール9は末尾サインオフを「質問などありましたら、XかLinkedinまでお願いします。それでは！」で固定している。本文は「質問などありましたら、XかLinkedinまでお願いします。」の後が「よいAgentic Analyticsを！」になっており、「それでは！」が無い。記事テーマに寄せた遊びとして「よいAgentic Analyticsを！」は著者性として悪くないが、固定形の「それでは！」を落とすかどうかは著者判断。

## 総評
最優先は2点。(1) 冒頭の順序（指摘1）— 宣言・年表先行を著者の現在地・体験スタートに組み替え、導入を軽くする。これが読者の入りを最も左右する。(2) 章構造とマークダウン階層の再構築（指摘3）— `##` 4章・`---` 区切りへ整え、説明的な見出しを短い宣言に削る。Wix 素抜けで階層と半角スペース（指摘4）が崩れている疑いがあるので、推敲冒頭でまとめて処理すると後工程が楽になる。事実面では版の断定（指摘6）を著者が一次確認するまで曖昧化しておくこと。
