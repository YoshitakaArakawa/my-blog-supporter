# メモ：態度としてのAI実践（別記事の種）

> **位置づけ**: 本編「なぜAI実践が進まないのか」の背景にある思想的な議論をまとめたもの。本編には直接載せないが、著者の思考の土台として保持する。将来の別記事の種、または本編を書く際に「自分の言葉で語るための下敷き」として使う。
>
> **重要な方針**: タレブ、パーシグへの名前出しの引用は本編ではノイズになる。概念は荒川自身の体験と言葉で語る。このメモは思考の裏側として残すが、そのまま記事にはしない。

---

## 1. 思考の土台：3つの源泉

### タレブ「身銭を切れ」——リスクを負わない助言は響かない
- 核：自分で痛みを引き受けてこそ本当の学びがある
- AI文脈での意味：
  - 「AIを使え」と外から言う人は、使わなかった場合のリスクを負っていない
  - 本人が自分の時間とキャリアを投じて初めて、意志は本物になる
  - 日本の環境ではリスクを負う理由がそもそも弱い → 本編の「外的圧力が弱い」論に溶かし済み
- **本編での使い方**: 名前を出さず、「自分の時間を投じて試すこと自体が学び」「頭で理解するのと実際に業務で試すのでは質が違う」として章3に溶かす

### パーシグ「禅とオートバイ修理技術」——仕事への態度がクオリティを決める
- 核：技術的な作業を「面倒な作業」と見るか「対象と向き合う行為」と見るかで、すべてが変わる
- AI文脈での意味：
  - AIを「効率化ツール」と見る人 → 効率化の必要がなければ使わない
  - AIを「思考の拡張」と見る人 → 必要性がなくても使う（仕事への態度そのもの）
  - 荒川さんの「ワクワク」「知的好奇心」の正体＝自分の仕事のクオリティに向き合う態度
- **本編での使い方**: 名前を出さず、「自分の仕事をもっと良くしたいという内発的な欲求」として章2の内的動機の説明に溶かし済み

### そーだいさんのブログ群——行動と学びのサイクル
- これは本編に参考資料として直接引用可能
- **特に重要な記事と論点**：

| 記事 | 核となる論点 | 本編への接続 |
|---|---|---|
| AI時代になぜ人が学ぶのか | 良い質問をするには知識が必要、自分で見つけた答えは忘れない | Why：AIを使いこなすにも基礎的な向き合いが必要 |
| 行動する技術 | ネクストアクション：具体的に、小さく、事実と感情を分ける。計画的偶発性 | 章3：小さく試す＝ガチャ試行の実践論 |
| 実績は土台を作って意思決定で積み上がる | 能力の土台×意思決定の積み重ね＝実績 | Why：意志→意思決定→実績の構造 |
| だって仕事でしょ と 所詮仕事でしょ | 仕事への向き合い方は善悪ではなく自覚の問題 | Why：内的動機の有無を断罪しない姿勢の根拠 |
| 一年の計は元旦にあり 2026年版 | AIで変わるのは学び方。アンラーニングの危機感 | Why：変わり続けること自体に内発的動機が要る |

## 2. 三者を統合して見える構造

**AI実践が進む人は「AIで何が効率化できるか」ではなく、「自分の仕事とどう向き合うか」の延長線上にAIがある。**

- リスクを負って試す（タレブ）→ 自分の言葉では「自分の時間を投じて小さく試す」
- 対象と向き合う態度（パーシグ）→ 自分の言葉では「自分の仕事をもっと良くしたいという欲求」
- 行動してサイクルを回す（そーだい）→ 自分の言葉では「ガチャ試行」「まず何か一つ投げてみる」

この3つは同じことを別の角度から言っている。本編ではこれらを荒川さん自身の体験として一つの語りに統合する。

## 3. 別記事として書く場合の方向性

### 仮タイトル案
- 「AI実践は効率化の話ではなく、仕事への態度の話だ」
- 「なぜ行動するのか——AI時代の学びと身銭」

### 想定読者
- 本編を読んで「意志はあるがどう向き合えばいいかわからない」と感じた人
- AI活用を推進する立場にいる人（マネージャー、トレーナー）

### 構成のイメージ
1. 仕事への2つの態度（こなすもの vs 向き合うもの）——パーシグ的な問い
2. 頭で理解するのと手を動かすのは違う——タレブ的な問い
3. 行動→内省→行動のサイクルをどう回すか——そーだい的な実践論
4. AI実践者は何に駆動されているのか——荒川自身の体験

### 執筆判断
- 本編の反応を見てから決める
- 本編がWhyの診断で完結しているので、こちらは「態度と行動」の処方箋的な位置づけになる
- Tableau NOTE の読者層を考えると、哲学書の引用は最小限にして体験ベースで語るのが良い
