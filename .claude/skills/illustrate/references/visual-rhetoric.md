# Visual Rhetoric（関係タイプ→視覚形）

## 使い方

作図の前に「この図で見せたい関係は何か」を **1つ** に絞る。複数の関係を1枚に詰めると論点がぼやける。絞った関係を下表で引き、推奨視覚形を採る。**「とりあえず四角を矢印で繋ぐ」は最後の手段**（プロセス/依存以外では情報量が出ない）。

## 対応表

| 見せたい関係 | 推奨視覚形 | 避けたい既定形 |
|---|---|---|
| プロセス・順序（A→B→C） | 横/縦のステップ列。各ステップに状態変化を1語添える | 等間隔の箱を矢印で繋ぐだけ（変化が読めない） |
| 一対多・分岐（1つから複数へ） | fan-out（中心ノードから放射）。枝の太さ/数で量を示す | 縦に箇条書きを並べる |
| 多対一・集約（複数が1つへ） | convergence（複数の流れが1点に収束） | 矢印を全部同じ終点に向けるだけの密集図 |
| 対比・二項対立（X vs Y） | 左右2カラムを軸（境界線/対称構図）で割る。同じ行で属性を対置 | 2つの箱を並べるだけ（差分が視覚化されない） |
| トレードオフ・程度（多いほど○、少ないほど×） | 2軸の散布/象限、またはスライダー/天秤。位置で度合いを示す | 表だけ（数値の大小が直感に乗らない） |
| 階層・包含（大→中→小） | 入れ子（同心/包含矩形）または木構造。深さ＝階層 | フラットなインデント箇条書き |
| before / after（変化の前後） | 左右2状態を同一構図で並べ、変わった箇所だけ強調 | 2枚を無関係に並置 |
| 構成要素の分解（全体→部品） | 分解図（exploded view）。全体像から部品を引き出す | 部品名を列挙するだけ |
| 因果・連鎖（原因→結果が連なる） | 一方向の連鎖。分岐/合流があれば node で表現 | 双方向矢印を多用して因果方向を曖昧にする |
| 時間推移・経過 | timeline（時間軸を1本通す）。出来事を軸上に配置 | 箇条書きの日付リスト |
| 重なり・共通部分 | ベン図/重なる領域。交差に意味ラベル | 別々の円を離して置く |
| 位置づけ・地図（全体の中のどこか） | マップ/座標上に対象をプロット。周囲との相対位置で語る | 単独の箱に名前を書くだけ |

## 選定の原則

- **図形に意味を持たせる**：枠の大きさ・位置・距離・太さ・色の濃淡が、関係の何か（量・順序・近さ・強さ）を表しているか。何も表していない装飾枠は削る。
- **1枚＝1主張**：図のキャプションを1文で書けるか。書けないなら関係を絞り直すか、図を分ける。
- **テキストは最小**：図は文章の代替ではなく、文章で説明しにくい関係を担う。ラベルは語/短句に留め、文を流し込まない。
- 表に当てはまらない関係なら、上の原則に立ち返って構図を自作してよい（表は出発点であって制約ではない）。
