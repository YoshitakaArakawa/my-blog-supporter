# Anti-Bland Rubric（凡庸さ判定）

判定は減点法だが、目標は減点回避ではなく **「飾った資料でなく、1枚の作品として成立しているか」**（雑誌の図版・ギャラリー作品の水準を北極星に置く。公式 canvas-design の "museum / magazine quality" 由来）。下の各項目は、その水準から外れる典型を潰すためのチェック。

## 採点手順

1. レンダリング済みの PNG を Read で開く（コードでなく**実際の見た目**を見る）。
2. 下の各項目を ✅/❌ で判定する。
3. ❌ が1つでもあれば作図に戻って直す。修正→再描画→再採点を繰り返す（**上限3周**）。
4. 3周しても残る ❌ は、サマリに「未解決」として正直に報告する（黙って合格にしない）。

## 採点項目

### A. 伝達（最優先）
- [ ] **3秒で主張が読めるか**：パッと見て「何を言いたい図か」が掴めるか。掴めないなら情報過多か焦点不在。
- [ ] **図形が意味を担っているか**：枠・位置・距離・大小・太さが関係（量/順序/近さ/強さ）を表しているか。ただの飾り枠が並んでいないか。
- [ ] **キャプション1文で言い切れるか**：言い切れないなら関係を絞り直す。

### B. スカスカ・凡庸の回避（今回の主目的）
- [ ] **「箇条書きを2次元に置いただけ」になっていないか**：箱に語を入れて等間隔に並べただけ、は不合格。
- [ ] **プレゼンスライド然としていないか**：タイトル＋中央寄せ箇条書き、の定番レイアウトに陥っていないか。
- [ ] **視覚的な階層があるか**：主役・脇役・補足の区別が、大きさ/濃淡/配置で付いているか。全要素が同じ重みだと平坦。
- [ ] **「ボックス＆矢印」への安易な回帰でないか**：プロセス/依存以外でこの形なら、visual-rhetoric.md で選び直す。

### C. 仕上がり
- [ ] **余白が機能しているか**：詰め込みすぎ（窒息）でも、スカスカで間が持たない（散漫）でもない。要素を意味のあるまとまりに群化できているか。
- [ ] **整列が取れているか**：エッジ/中心が揃っているか。微妙にズレた要素は素人っぽさの主因。
- [ ] **テキスト量が適切か**：図に文章を流し込んでいないか。ラベルは語/短句。読ませる説明は本文側に置く。
- [ ] **切り出しが正しいか**：要素が見切れていない／意図しない余白が大きすぎない。直すなら SVG の viewBox を要素に合わせるか、screenshot の target を `"svg"` 要素に絞る。
- [ ] **同梱フォントで描かれているか**：豆腐（□）やシステム既定フォントへのフォールバックに化けていないか。`document.fonts.ready` の status が全て `loaded` か（化けていたら base64 埋め込み手順を確認する）。

## 不合格時の典型処方

- 主張が読めない → 要素を削る。最重要の1関係以外を落とす。
- 平坦 → 主役を1.5〜2倍に、脇役を縮小・淡色化して階層を作る。
- スライド然 → 中央寄せ箇条書きをやめ、visual-rhetoric.md の関係特化の構図に置換。
- 窒息 → 余白を増やし、要素を2〜4群に群化。
- 見切れ/余白過多 → SVG の viewBox を要素に合わせ直すか、screenshot の target を `"svg"` 要素に絞る。
