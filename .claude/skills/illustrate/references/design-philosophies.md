# Design Philosophies（複数案の分散と品質）

## 大原則：分散は「仕込む」もの

同じ指示で N 体走らせると似た絵に収束する。各変種に**別々の philosophy（美学方針）**を割り当てて初めて分散が出る。**内容（図にする主張・要素の意味対応）は全変種で同一に固定し、変えるのは表現（medium・構図・比喩・配色・タイポ）だけ。**

## philosophy レンズ集（変種ごとに1つ割り当てる）

| レンズ | 狙い | 典型の medium / 表現 |
|---|---|---|
| 技術構造図 | 正確さ・明快さ。エンジニアのホワイトボード | SVG。レーン/コンテナ境界、monospace アクセント、request/response ループ |
| 比喩イラスト | 親しみ・記憶に残る。概念を物として描く | SVG or HTML。象徴アイコン、温かい配色、手描き感 |
| 抽象幾何 | ポスター的・洗練。図形と余白と色で語る | SVG。最小要素、大胆な構図、限定パレット |
| ドメインメタファー | 概念を現実の場面に翻訳（盤面・調理場・機械・操り人形 等） | HTML or SVG。シーンを1枚絵にする |
| タイポグラフィック | 言葉そのものを主役に。型で意味を出す | HTML/CSS。大きな型、実フォント、最小の図 |
| フロー/シーケンス | 動き・順序・往復を見せる | SVG。矢印の流れ、段階、タイムライン |

レンズは出発点。当てはまらなければ原則（下記）に従って独自の美学を立ててよい。

## 純コード描画で質を上げる技法（公式デザインスキル由来）

50点を超えるための実務レバー。画像生成モデルは使わない前提での質の作り方：

1. **philosophy-first**：描く前に「この絵の美学」を1段落で言語化する（名前・原則・なぜこの概念に合うか）。レイアウトでなく美学を先に決める。
2. **medium を選ぶ**：
   - **SVG** … 図・幾何・正確な座標。PNG 書き出しが綺麗。
   - **HTML/CSS** … gradient・影・blend・リッチなレイアウト・実フォントが要るとき。chromium が高品質に描く。**同じ playwright 描画パイプで撮れる**（SVG をラップする代わりに HTML 全体を data URL で開く）。
3. **実フォントを使う**：システム既定に頼らない。HTML medium なら `@import`/`@font-face` で Web フォントを読む（日本語は Noto Sans JP / Zen Kaku Gothic 等）。タイポの質は第一印象を大きく左右する。フォント読込後に撮るため、`playwright-cli eval "document.fonts.ready.then(()=>document.fonts.size)"` で待ってから screenshot する。
4. **構図と余白**：グリッドで整列、余白を惜しまない、焦点の階層（主役を大きく・濃く）を明確に。
5. **抑制**：パレットは 2〜3 色＋ニュートラル。テキストは最小。装飾枠を足さない。
6. **second-pass で磨く**：自己レビュー後、バグ修正だけで終わらせず「もう一段良くする」推敲を1回入れる（間延び・凡庸さの除去）。

## 変種を作るときの固定/可変

- **固定（全変種で同一）**：図にする主張、要素の意味対応（例: 脳＝X／手＝Y）、事実関係・固有名詞。
- **可変（変種ごとに変える）**：philosophy レンズ、medium、構図、比喩、配色、タイポ、余白の使い方。
