# import-wix-draft

Wixブログ編集ページを丸ごと保存したMHTMLから、記事本文だけを抽出して `draft_user.md` に書き出す。AIと「現在の下書き」を共有して議論するための取り込み工程。

**この Skill は抽出までを担う。** 書き出したら実行サマリ（タイトル・文字数・保存先）を報告して終わる。議論・推敲はユーザーが別途切り出す。

## 前提

- Python 3（標準ライブラリ `email` / `html.parser` のみ。追加パッケージ不要）。
- 入力は Wix ダッシュボードのブログ編集ページを保存した MHTML。内部の記事プレビュー（`single-post` のレンダリング結果）に本文が含まれる。

## 抽出方式

`scripts/extract_wix_mhtml.py` が以下を機械的に行う（HTML解析が脆いので、このスクリプトを正確に実行する）：

- MHTML を MIME multipart として解析し、全 `text/html` パートを UTF-8 デコード
- 各パートから `data-hook="post-description"` コンテナをサブツリー抽出し、最長のものを本文に採用（どのパートに本文があるかをハードコードしない）
- `data-hook="post-title"` をタイトルに採用
- depth 追跡＋void要素対応でコンテナ境界を厳密に判定し、サイトのナビ・カテゴリ・フッターを構造的に排除
- ブロック単位でテキスト化し、段落・見出し・区切り線は空行で区切って書き出す（連続するリスト項目だけは詰める）

### 構造の復元範囲

Wix の DOM に**実在する構造**だけを Markdown に変換する。フォントサイズ等からの見出し推測はしない（推測でマークアップを足さない）。

- 見出し：`<h1>`〜`<h6>` を `#`〜`######` に変換（Wix の編集プレビューは見出しを意味タグで吐く）
- 区切り線：`type="divider"` の空 div を `---` に変換
- リスト：`<li>`（内側に `<p>` を持つ Wix 構造を含む）を `- ` 行に変換
- 整形済みブロック：`<pre>` 内のリテラル改行を保ち、行ごとに段落へ分割（公開日や複数行メモが1行に潰れるのを防ぐ）

リンクの URL は Wix プレビューでは復元できないことがあり、その場合はリンクテキストを素のテキストとして残す（URL を捏造しない）。

## 手順

進捗：
- [ ] ステップ1：入力 MHTML を特定する
- [ ] ステップ2：出力先 draft_user.md を決める
- [ ] ステップ3：上書き確認
- [ ] ステップ4：抽出スクリプトを実行する
- [ ] ステップ5：サマリを報告する

### ステップ1：入力 MHTML を特定する

- 引数でパスが渡されたらそれを使う。
- 引数が無ければ Glob `output/**/*.mhtml` で探し、最新（mtime 最大）を採用する。複数候補があり最新が意図と異なりそうなら、候補を提示して1問だけ確認する。
- 1件も無ければ、その旨を報告して終わる（勝手に他の場所を探さない）。

### ステップ2：出力先 draft_user.md を決める

入力 MHTML と同じディレクトリ（記事の出力ディレクトリ `output/{yyyymmdd}_{テーマ}/`）の `draft_user.md` を出力先にする。

### ステップ3：上書き確認

`draft_user.md` が既に存在し中身が非空なら、上書き前にユーザーへ確認する。`draft_user.md` はワークフロー上「ユーザー推敲版」のスロットで、既存の手書き内容を失う恐れがあるため。空ファイルまたは未作成ならそのまま進む。

### ステップ4：抽出スクリプトを実行する

```
python .claude/skills/import-wix-draft/scripts/extract_wix_mhtml.py "<入力.mhtml>" "<出力ディレクトリ>/draft_user.md"
```

スクリプトは成功時に `OK title=... / OK output=... / OK body_chars=... body_lines=... headings=...` を出力する。`error:` で終わった場合は、メッセージをそのままユーザーに伝え、勝手な代替経路（別パートの当て推量抽出など）に逃げない。

### ステップ5：サマリを報告する

`OK` 行をもとに、抽出タイトル・本文文字数・見出し数・保存先パスを報告する。本文全文はここで貼らない（ファイルにある）。議論したい場合はユーザーが続けて指示する。
