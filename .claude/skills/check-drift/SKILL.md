# 段階間ドリフト点検

AI が生成した中間成果物（`outline.md` / `draft.md`）が、その前段の成果物から **主張を落としていないか**・**前段に根拠のない主張を足していないか** を点検し、優先度順の指摘を `output/{yyyymmdd}_$ARGUMENTS/drift/drift_NN.md` に出力する。

**フォルダ命名規則**: 出力先は `output/{yyyymmdd}_$ARGUMENTS/` とする。既存フォルダがあればそれを使う。`$ARGUMENTS` に日付プレフィックス付きのフォルダ名が直接渡された場合はそのまま使う。以降のパス記述で `$OUTPUT_DIR` はこのディレクトリを指す。

このスキルは隔離コンテキスト（`context: fork`）で実行される。会話履歴にはアクセスできないため、点検対象と基準はすべてファイルから読み込む。書いた当人（write/outline の対話）から切り離された別エージェントが「文書として」読み比べることで、**自分が過剰追加した主張を自分で擁護する経路を断つ**。これは critique を deepen から、review を本文執筆から分離したのと同じ思想。メイン会話には**実行サマリのみ**を返す。指摘の全文を戻り値に詰めない。

## このスキルの役割

掘って固めた思考（thinking.md）や構成（outline.md）が、AI の手で散文化・縮約される過程で静かにズレるのを、工程の切れ目で捕まえる専任。狙いは「ずれたまま次工程へ持ち越さない」こと（何を見るかは手順3で定義する）。

工程内での役割分担を守る（重複させない）:

- **critique**: thinking.md の**主張そのものの是非**を叩く（想定反論・前提の穴・論理の飛躍）。本スキルは主張の是非を問わない。前段にあった主張が後段で**保たれているか**だけを見る。
- **review-author-draft**: **著者が書いた** draft_user.md の編集品質を診る。本スキルは **AI が生成した** outline.md / draft.md が対象。
- **compare-drafts**: draft.md（AI版）と著者の公開記事を比較する。本スキルは AI 成果物の**前段との整合**であって、著者版との比較ではない。

ここで出た指摘は、著者が「この差分を許容するか／前段に戻して直すか」を決めるための材料。**成果物を書き換えない**。提案までに留め、本文への直接編集はしない。

## 点検する遷移

遷移ごとに「健全な動き」が違うため、基準と target を切り替える。`$ARGUMENTS` の末尾トークンが `outline` または `draft` ならそれを **stage** とし、残りをテーマとする。stage 省略時の既定は `draft`。

| stage | 基準（前段の正） | target（点検対象） | 主に見るドリフト |
|---|---|---|---|
| `draft`（既定） | `thinking.md`（主張の正）＋ `outline.md`（構成の正）＋ `references.md`（根拠源） | `draft.md` | 落とした背骨／無根拠に湧いた主張 |
| `outline` | `thinking.md`（主張の正）＋ `brief.md`（コアメッセージの正） | `outline.md` | 縮約で落ちた背骨／構成段階で湧いた新主張 |

`brief.md → thinking.md` の遷移は点検しない。掘る工程は主張が増えるのが正常で、add を咎める点検は低価値なため。叩きたい場合は critique を使う。

## 手順

### 1. stage とテーマの確定

`$ARGUMENTS` を解釈する:

- 末尾トークンが `outline` / `draft` なら stage に採り、残りをテーマ名にする
- 末尾トークンが stage キーワードでなければ stage は既定の `draft`、`$ARGUMENTS` 全体をテーマ名にする

確定した stage に応じて、上表の基準ファイルと target ファイルを以降で使う。

### 2. 入力の読み込み

`$OUTPUT_DIR` から以下を読む:

- **target ファイル**（必須）: stage が `draft` なら `$OUTPUT_DIR/draft.md`、`outline` なら `$OUTPUT_DIR/outline.md`
- target が無い・空の場合は、点検せずに「target が見つからない（または空）ので、先に該当工程を回してください」という旨をサマリで返して終了する。**他ファイルで代替しない**
- **基準ファイル**: 上表の前段の正。stage=`draft` なら `thinking.md`＋`outline.md`＋`references.md`、stage=`outline` なら `thinking.md`＋`brief.md`。**`thinking.md` は必須**（主張の正がなければドリフトを測れない）。無ければサマリで欠落を報告して終了する。`outline.md` / `references.md` / `brief.md` は無ければ「無し」として進める
- `$OUTPUT_DIR/drift/` 内の既存 `drift_*.md`（あれば。前回までに指摘済みのドリフトを把握し、解消済み／未解消に印を付ける）

### 3. ドリフトの抽出

基準（前段の正）に現れる**主張ユニット（idea）**と、target に現れる主張ユニットを、表現の長短を無視して **idea 単位**で対応付ける。1段落＝1主張とは限らない。対応付けた結果から、次の2軸だけを拾う。

**A. 落とした（基準にあり target に無い）**
各 idea を次のどちらかに判定する:
- **背骨落ち ⚠**: コアメッセージ・主要な論点・主張の核を成す idea が target から消えている。最優先で報告する
- **意図的剪定（ok）**: 枝葉・重複・脇道で、落ちても主張は痩せない。健全な縮約。簡潔に列挙するに留める

判定基準は brief.md / thinking.md のコアメッセージ。それを支える idea が落ちていれば背骨落ち。

**B. 足した（target にあり基準に無い）**
各 idea を次のどちらかに判定する:
- **無根拠 add ⚠**: 前段（thinking.md / outline.md / references.md）に根のない主張が target で湧いている。AI が著者の判断を経ずに中身を差し込んだ疑い。**どこにも根が無いことを明示**して最優先で報告する
- **根拠あり（grounded）**: references.md の資料や thinking.md の含意から自然に導ける肉付け・例示・接続。どの基準ファイルに根があるかを示す

散文化に伴う言い換え・具体例の補足それ自体は咎めない。**前段に無い新しい主張・評価・断定**が入っているかを見る。

**注意**:
- 定量指標（carryover 率・件数比など）は出さない。著者が知りたいのは *どの* idea がドリフトしたかであって率ではない
- 主張の正しさは判定しない（それは critique）。あくまで「前段から保たれたか／前段に根があるか」だけを見る

### 4. 連番の決定

`$OUTPUT_DIR/drift/` 内の既存 `drift_NN.md` を確認し、最大番号 + 1 を今回の番号とする（2桁ゼロ埋め）。1つも無ければ `drift_01.md`。フォルダが無ければ作成する。再点検（直して回し直した後）の変遷が時系列で残る。

### 5. drift_NN.md の生成

以下のフォーマットで `$OUTPUT_DIR/drift/drift_NN.md` に書き出す:

```markdown
# ドリフト点検 #{NN}（{yyyymmdd}）

## 点検した遷移
{基準} → {target}（stage: {draft|outline}）

## 基準のコアメッセージ
{thinking.md / brief.md から読み取ったコアメッセージを一行で}

## 前回からの変化
{drift_01 等が既にある場合、前回の ⚠ が解消されたか／未解消か。初回は「初回点検」}

## ⚠ 要確認（優先）

### 落とした背骨
- **{落ちた主張の見出し}** — {基準のどこにあったか} / {target で消えている理由の推測} / {戻すべきか剪定でよいかの方向性}

### 無根拠に足した主張
- **{湧いた主張の見出し}**（{target}:行番号） — {前段のどこにも根が無いことの確認} / {残すなら根拠を補うか、著者判断を仰ぐべき点}

## 健全な差分（参考・簡潔に）

- 意図的剪定: {落ちても主張が痩せない idea を短く列挙}
- 根拠ある肉付け: {湧いたが基準に根がある add を短く列挙、根の在処つき}

## 総評
{著者が最優先で確認すべきドリフトを1〜2個。戻すべきものと許容してよいものの線引き}
```

`{yyyymmdd}` は実行日の日付。⚠ が1件も無い場合は「要確認なし（前段から逸脱していない）」と明記する。

### 6. 実行サマリを返す

メイン会話には以下だけを返す:
- 書いたファイルパス（`$OUTPUT_DIR/drift/drift_NN.md`）
- 点検した遷移（基準 → target）
- ⚠ の有無と、最優先で確認すべきドリフト（1〜2行。背骨落ち／無根拠 add を具体に）

著者がこの1〜2行で「次工程へ進む／前段に戻して直す」を判断できるようにする。指摘の全文は返さない。Write に失敗した場合は inline で指摘本文を返さず、失敗した旨をサマリで返す。
