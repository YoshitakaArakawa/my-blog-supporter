# 批判レビュー

`output/{yyyymmdd}_$ARGUMENTS/thinking.md` に書かれた主張を批判的にレビューし、想定反論・前提の穴・論理の飛躍を指摘して `output/{yyyymmdd}_$ARGUMENTS/critique/critique_NN.md` に出力する。

このスキルは隔離コンテキスト（`context: fork`）で実行される。会話履歴にはアクセスできないため、レビュー対象はすべてファイルから読み込む。メイン会話には**実行サマリのみ**を返す（書いたファイルパス・指摘件数・主要論点）。指摘の全文を戻り値に詰めない。

## このスキルの役割

deepen が「著者の言葉を引き出して掘る」のに対し、critique は「掘った主張を叩く」専任。引き出す工程と叩く工程をコンテキストごと分離することで、メインの deepen 対話は最後まで「引き出す人格」を保てる。

ここで出た指摘は、次の brainstorm⇄deepen ループに戻して深掘りの燃料にする。批判すること自体が目的ではなく、思考を次の段に進めることが目的。

## 手順

### 1. 入力の読み込み

`$OUTPUT_DIR`（`output/{yyyymmdd}_$ARGUMENTS/`）から以下を読む:

- `$OUTPUT_DIR/thinking.md`（必須。レビュー対象の主張メモ）
- thinking.md がなければ、ファイルが見つからない旨をサマリで返して終了する（inline で代替出力しない）
- `$OUTPUT_DIR/brief.md`（あれば。テーマ・動機・コアメッセージの文脈把握に使う）
- `$OUTPUT_DIR/references.md`（あれば。主張と参考資料の整合を見るのに使う）
- `$OUTPUT_DIR/critique/` 内の既存 `critique_*.md`（あれば。前回までに何を指摘済みか把握し、重複指摘を避ける）

### 2. 批判的レビュー

thinking.md の主張・コアメッセージ・各観点を、以下のレンズで多角的に叩く。1つのレンズに偏らず一通り当てること。

- **想定反論**: 読者や逆の立場の人が「でも〜では？」と返しそうな点。最も強い反論を具体的に立てる
- **前提の穴**: 主張が暗黙に置いている前提のうち、検証されていない・崩れうるもの
- **論理の飛躍**: 「Aだから B」の間に飛躍がある箇所。根拠と結論の距離
- **コアメッセージの弱さ**: 主張が当たり前すぎないか、逆に飛躍しすぎていないか。読後に何が残るか
- **既出との重複**: 前回までの critique で指摘済みの論点は繰り返さず、未対応なら「未対応」と印を付ける

各指摘には「なぜそれが問題か」と「次のループでどこを掘り直すとよいか」をセットで書く。叩いて終わりにせず、次の一手に繋げる。

**やらないこと**:
- 著者の主張を別の主張で置き換える提案（それは deepen/outline の領分）
- 文章表現・推敲レベルの指摘（それは write の領分）
- 既存記事との重複チェック（yarakawa.com 側のデータ取得が前提のため、このスキルの初版では扱わない）

### 3. 連番の決定

`$OUTPUT_DIR/critique/` 内の既存 `critique_NN.md` を確認し、最大番号 + 1 を今回の番号とする（2桁ゼロ埋め）。1つも無ければ `critique_01.md`。フォルダが無ければ作成する。

これにより過去の指摘が上書きで消えず、ループごとの変遷が時系列で残る。

### 4. critique_NN.md の生成

以下のフォーマットで `$OUTPUT_DIR/critique/critique_NN.md` に書き出す:

```markdown
# 批判レビュー #{NN}（{yyyymmdd}）

## レビュー対象
{この時点の thinking.md のコアメッセージを一行で}

## 前回からの変化
{critique_01 等が既にある場合、前回指摘がどう反映されたか／未対応か。初回は「初回レビュー」}

## 指摘

### 1. {指摘の見出し}
- レンズ: {想定反論 / 前提の穴 / 論理の飛躍 / コアメッセージ}
- 問題: {何がどう弱いか}
- 次の一手: {次のループでどこを掘り直すか}

### 2. {指摘の見出し}
...

## 総評
{今回のループで最も優先して掘り直すべき論点を1〜2個}
```

`{yyyymmdd}` は実行日の日付。

### 5. 実行サマリを返す

メイン会話には以下だけを返す:
- 書いたファイルパス（`$OUTPUT_DIR/critique/critique_NN.md`）
- 指摘件数
- 最優先で掘り直すべき論点（1〜2行）

指摘の全文は返さない。Write に失敗した場合は inline で指摘本文を返さず、失敗した旨をサマリで返す。
